## 작업 이력 (날짜순)

### 2026-06-07 — 전체 개발 및 배포 완료

#### 1단계: 엑셀 파일 구조 분석
- `상품정보DB/` 폴더에 23개 `.xls` 파일 확인
- 모든 파일 동일 구조: **55개 시트**, 파일당 상품코드 1개
- **핵심 발견**: 헤더는 `header=23`(0-indexed 행 23), DATA 레이블 행부터 실제 데이터 시작
- 주요 시트: `상품`(113컬럼), `상품담보`(77컬럼), `상품판매기간`(7컬럼)
- 담보 수: 상품당 31개(실손) ~ 2,284개(건강쑥쑥 어린이)

#### 2단계: 전체 앱 구조 개발
- Flask + PostgreSQL + xlrd 구조 확정
- pandas 대신 **xlrd 단독 사용** (Render 빌드 오류 해결)
- DB 테이블 3개: `products`, `coverages`, `file_upload_log`

#### 3단계: Render 배포 과정 (트러블슈팅)
- 문제1: Render가 Python 3.14 사용 → pandas 빌드 실패
- 해결: `PYTHON_VERSION=3.11.9` 환경변수 + pandas 완전 제거
- 문제2: `google-generativeai==0.8.5` 버전 없음 → 빌드 실패
- 해결: 버전 핀 제거
- 문제3: Gemini API `gemini-1.5-flash` 404 → `gemini-2.0-flash-lite` 변경
- 문제4: Gemini 무료 티어 `limit: 0` → 챗봇 임시 비활성화

#### 4단계: UI 개선
- 업로드 진행 바 추가 (파일별 AJAX, 실시간 표시)
- 상품 목록 상세 버튼 주황색으로 변경
- 상품판매기간 컬럼 추가 (상품판매기간 시트 파싱)
- 담보 목록 독립특약여부·독립특약상품코드 컬럼 추가
- 담보 목록 컬럼별 필터 기능 (텍스트 검색 + 드롭다운)
- 상단 타이틀 변경: "장기보험 상품 너의 모든것이 궁금해....!'"

## Render 환경 설정

| 항목 | 값 |
|---|---|
| 플랫폼 | Render.com 무료 플랜 |
| Python 버전 | 3.11.9 (환경변수 PYTHON_VERSION으로 고정) |
| DB | PostgreSQL (Render 무료, 90일 만료 주의) |
| 슬립 모드 | 15분 미사용 시 절전, 첫 접속 30~60초 대기 |
| Build Command | pip install -r requirements.txt |
| Start Command | gunicorn app:app |

### 환경변수 (Render → Environment 탭)
- `DATABASE_URL`: Render PostgreSQL Internal URL
- `GEMINI_API_KEY`: Google AI Studio API 키
- `PYTHON_VERSION`: 3.11.9

## 향후 추가 예정 기능

### AI 챗봇 복구
- Gemini API 무료 티어 문제 해결 필요
- aistudio.google.com에서 "Create API key in new project"로 순수 AI Studio 키 발급
- 개인 Gmail 계정으로 발급 시 무료 티어 정상 작동 가능성 높음

### 정합성 체크 고도화 (담당 업무 핵심)
- 동일 상품 종(種) 간 담보 자동 비교 리포트 (LA02928001~008)
- 개정 전/후 담보 변경 내역 자동 추출
- 담보코드 동일 + 담보명 다른 케이스 탐지 (오기입 탐지)
- 필수 담보 누락 체크 기능

### 상품 상세 화면
- 기본 정보 항목 추가 예정 (현재 상품코드, 인가명만 표시)

### DB 만료 대응 (90일 후)
- Render PostgreSQL 무료 DB는 90일 후 만료
- 만료 전 유료 전환($7/월) 또는 DB 재생성 후 엑셀 재업로드 필요
